# Backend Rules

Path-scoped rules for `host/**` (Python orchestrator) and `guest/**`
(Rust NT service). The high-level "Coding rules" in
[AGENTS.md](../../AGENTS.md) is the canonical statement; this file
restates and adds the path-specific guardrails universals expects.

## Host (Python 3.9+) — `host/**`

Stack: Python 3.9 minimum; asyncio end-to-end; mypy `--strict`;
`black` formatting; pytest; gRPC over `AF_VSOCK` with mTLS.

- **Asyncio everywhere.** No `while True: sleep(…)` polling. If you
  need a timer, use `asyncio.sleep` inside a task; if you need to
  observe state changes, use a stream or a `Future`. Anything else
  is a smell.
- **Full type hints; `mypy --strict` is the gate.** Pre-commit hook
  runs `mypy --strict src/` whenever a `host/**.py` file changes.
  Keep `Any` out of public surfaces; if you must use it, comment
  why on the same line.
- **Validate at boundaries, not internally.** gRPC servicer entry
  points and libvirt response parsers validate. Internal helpers
  trust their callers — no defensive re-checking of arguments that
  came from another module in this package.
- **HTTP / gRPC client timeouts are mandatory.** Never start a
  request without an explicit timeout. `aiohttp.ClientTimeout`,
  gRPC `timeout=` keyword, libvirt event-loop deadlines — pick one.
  Infinite hangs are bugs, not edge cases.
- **No retry on 4xx.** Retry only on 5xx, network errors, or
  explicit transient gRPC codes (`UNAVAILABLE`, `DEADLINE_EXCEEDED`).
  4xx is "you asked wrong" — fix the call site instead.
- **Tests:** unit tests in `host/tests/` colocated with what they
  test; integration tests that need libvirt or a real VM use a
  pytest marker so they don't run by default. Keep the smoke
  in-process suite (`test_smoke_inprocess.py`) green at all times.
- **Generated proto stubs:** `host/src/crossdesk_host/proto/*_pb2*.py`
  are regenerated by `python build_proto.py`. Don't edit them by
  hand; the manifest in `.claude/ignorefiles.md` flags them.

## Guest (Rust) — `guest/**`

Stack: Rust on Windows NT (cross-compiled `x86_64-pc-windows-gnu`
for production; `cargo check --workspace` on dev host for sanity);
`tonic` for gRPC; `windows-rs` for NT service plumbing.

- **Idiomatic Rust.** `unwrap()` / `expect()` need a one-line
  `// Safety:` or `// Infallible because: …` comment immediately
  above explaining why the call is okay. No silent `unwrap`s.
- **Pre-commit: `cargo check --workspace --quiet` for `guest/`**
  (and `cargo check --quiet` for `gui/`). The hook only runs the
  relevant check based on which paths changed.
- **Pre-push: `clippy` clean.** The pre-push hook doesn't run
  clippy directly (cost vs. push frequency tradeoff), but the CI
  workflow does, so don't push code that would fail clippy.
- **Generated proto types** under `guest/crates/proto/` come from
  `tonic-build`; don't edit by hand.

## Proto-first RPC pattern

Any new RPC, regardless of language:

1. Edit `proto/crossdesk/v1/<service>.proto` (single source of
   truth — wire-format changes need owner approval per
   [AGENTS.md](../../AGENTS.md) "File boundaries").
2. Regenerate stubs: `python build_proto.py` for host, `cargo
   build` for guest (tonic regenerates as part of the build).
3. Implement the servicer in
   `host/src/crossdesk_host/ipc/<service>.py`.
4. Wire it into the watchdog FSM in
   `host/src/crossdesk_host/watchdog/` if the new RPC affects
   lifecycle (Hello, heartbeat, suspend/resume, hard-destroy).

## Secrets & mTLS

- **Every API token via env var or a gitignored `.env`.** Never
  hardcoded.
- **mTLS leaf certs** under `infra/certs/` are gitignored (`.gitignore`
  already configured). The CA root and the generation script
  (`generate_mtls.sh`) are tracked; the leaves are not.
- **Per-frame `AuthContext`** (peer cert fingerprint + stream nonce
  + monotonic sequence) is the second line of defense beneath
  mTLS — it must be checked on every frame, not just the first.
  Touching this code requires updating
  `docs/THREAT_MODEL.md` (per AGENTS.md "File boundaries").
